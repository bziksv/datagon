Краткие определения сущностей и процессов, которые встречаются в панели и в [REST API](/docs/api/).

## Конкуренты и парсинг

| Термин | Что это |
|--------|---------|
| **Проект (конкурент)** | Запись в разделе [Конкуренты](/docs/projects/): сайт конкурента + **CSS-селекторы** для вытягивания цены, названия, SKU и т.д. с HTML-страницы карточки товара. |
| **Очередь (`pages`)** | Список **URL**, которые нужно скачать и разобрать парсером. Статусы: ожидание, успех, ошибка и др. Экран [Очередь](/docs/queue/). |
| **Парсинг** | HTTP-запрос к странице конкурента + разбор HTML по селекторам + запись строки в **`prices`** (результаты). |
| **Результаты (`prices`)** | Таблица собранных цен/наличия по конкурентам. Экран [Результаты](/docs/results/). |

## Наш каталог

| Термин | Что это |
|--------|---------|
| **Мой сайт (источник)** | Подключение к **вашей** внешней БД (Bitrix, Webasyst и т.п.). Экран [Мои сайты](/docs/mysites/). |
| **`my_products`** | Локальная таблица в БД Datagon: «наши» товары, подтянутые из источников. Экран [Мои товары](/docs/myproducts/). |
| **`is_active`** | Признак того, что позиция **есть в последнем синке** источника. Перед синком записи помечаются неактивными; найденные в источнике снова становятся активными. |
| **`source_enabled`** | Признак на стороне источника (включён/выключен товар в вашей витрине) — используется в фильтрах. |

## Сопоставление и МойСклад

| Термин | Что это |
|--------|---------|
| **Сопоставление (матчинг)** | Поиск пар «наш товар ↔ товар конкурента» по SKU и/или по названию с оценкой похожести. Экран [Сопоставление](/docs/matches/). |
| **Кандидат / pending** | Найденная пара, ожидающая ручного **подтверждения** или **отклонения**. |
| **`ms_export`** | Локальная таблица выгрузки номенклатуры из **API МойСклад**. Экран [МойСклад](/docs/moysklad/). |
| **Связь с МС** | В «Моих товарах»: есть ли соответствие кода товара строке в `ms_export` (нормализация при синке). |

## Система

| Термин | Что это |
|--------|---------|
| **Логи / процессы** | Сводка **фоновых задач**: глобальный синк, МойСклад, парсинг, матчинг. Экран [Логи](/docs/processes/). |
| **Настройки** | Глобальные параметры парсинга, синхронизации, токены, пользователи. [Настройки](/docs/settings/). |
| **Веб-справка** | Этот сайт на Docusaurus: исходники `docs-docusaurus/docs/`, сборка в `public/docs/`, URL **`/docs/`**. |
| **Воркер (`worker.js`)** | Отдельный процесс (или сервис), который забирает `pages` в обработку и пишет в `prices`; без него очередь «не двигается». |
| **Discover** | Автодобавление URL в очередь по sitemap/правилам; лимиты `discover_*` в настройках. |
| **`dg_session` / cookie** | Серверная сессия для доступа к `/docs/*` и панели; см. `session-bridge.html` в статике доков. |
| **Нормализация SKU** | Приведение артикула к каноническому виду (пробелы, регистр) при сравнении `my_products` и `ms_export` / конкурента — логика на сервере. |
| **Порог матчинга** | Число `threshold`: минимальная «уверенность» пары наш–чужой. |

Если термин в интерфейсе не совпадает с таблицей — ориентируйтесь на подсказки (иконки «?») и на названия колонок в таблицах.
