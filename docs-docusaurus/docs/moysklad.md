## С чего начать новому пользователю

1. Убедитесь, что в `config.js` или в переменной окружения **`MS_TOKEN`** задан рабочий токен с правами на чтение нужных сущностей МС.
2. Откройте **МойСклад** и проверьте **статус** последней задачи (не висит ли ошибка с прошлого запуска).
3. При необходимости нажмите **Синхронизация** и дождитесь завершения (индикатор в UI + [Логи](/docs/processes/)).
4. Используйте **поиск** и фильтры, чтобы найти конкретный **код/артикул** и сверить поля с вашей учётной политикой.

## Блоки интерфейса

| Блок | Назначение |
|------|------------|
| Статус / прогресс | Отображение фоновой задачи синка с API МС; текст ошибки при сбое. |
| Фильтры | Тип (`Товар` / `Комплект` / «все»), архив, остатки, «на сайте», строка поиска. |
| Таблица | Строки `ms_export`; широкая таблица со скроллом; настройка видимых колонок (если есть). |
| Действия | Запуск/остановка синка, пересборка кэшей (при наличии). |

## Типовой сценарий сверки с «Моими товарами»

В [Мои товары](/docs/myproducts/) признак **связи с МС** опирается на сопоставление **нормализованных** кодов между `my_products` и `ms_export`. Если связь «плавает»:

- выполните повторный **sync** МС;
- проверьте, что в МС и у вас в источнике **одинаковая логика** артикула (пробелы, регистр нивелируются на сервере — см. API);
- при массовых правках — кэш статистики может задержаться на TTL.

## Если синхронизация не идёт

1. [Логи](/docs/processes/) — не застряла ли предыдущая задача.
2. Токен и лимиты API МС (ошибки 401/403 в логах сервера).
3. `GET /api/ms/status` — вручную через API (см. [документацию API](/docs/api/#moysklad)).
4. После смены токена в конфиге — **перезапуск Node**.

## Колонки `ms_export` (общий смысл)

Набор колонок зависит от версии выгрузки; ориентиры:

| Тема | Что смотреть |
|------|----------------|
| **Код / артикул** | Связь с [Мои товары](/docs/myproducts/) после нормализации. |
| **Тип** | Товар / **комплект** — комплекты часто требуют отдельной бизнес-логики в отчётах. |
| **Остаток, цены** | Сверка с витриной МС; расхождения — часовой пояс и налоги. |
| **«На сайте» / архив** | Фильтры в UI; помогают отсечь неактуальные строки при ручной проверке. |

## Лимиты API МойСклад

У МС есть **rate limit** и размер страницы; в Datagon см. `ms_sync_page_limit` и `ms_sync_delay_ms` в [Настройках](/docs/settings/). При `429` или таймаутах — **увеличьте** паузу, **уменьшите** страницу, не запускайте второй синк, пока первый не завершён.

## API (для автоматизации)

- `POST /api/ms/sync` — старт фона.
- `GET /api/ms/status` — состояние.
- `GET /api/ms/export` — страницы таблицы с query `search`, `type`, `limit`, `offset`.

Полное описание — [REST API — MoySklad](/docs/api/#moysklad).
