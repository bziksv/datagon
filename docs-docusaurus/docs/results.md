## Как читать строку результата

- **Проект** — какой конкурент (см. [Конкуренты](/docs/projects/)).
- **Время** — когда парсер завершил запись (часовой пояс — как настроен на сервере/в БД).
- **Цена / SKU / Название** — извлечённые значения; пустое поле чаще всего означает промах селектора или отсутствие узла в HTML.

## Фильтрация и сортировка

Используйте фильтры по **проекту**, диапазону дат, текстовому поиску — набор зависит от версии. **Сортировка** по колонкам помогает найти последние изменения или самые большие расхождения.

## Действия

- **Повторный парсинг** выбранных строк — после правки селекторов в проекте.
- **Очистка** результатов (глобально или по проекту) — **необратима** в смысле удаления строк из `prices`; используйте осознанно.

## Колонки и поля (ориентир)

Точный набор колонок зависит от версии UI, но логика типична:

| Колонка / поле | Зачем смотреть |
|----------------|----------------|
| **Проект** | Связь с [Конкурентами](/docs/projects/); фильтр «только этот сайт». |
| **URL / страница** | Какая карточка была разобрана; помогает найти кривой селектор. |
| **SKU / артикул** | Качество сопоставления с «нашими» товарами; пусто — проверить селектор SKU. |
| **Цена, валюта** | Сырые значения после Cheerio; неверная валюта — отдельное поле в проекте. |
| **Время** | Порядок прогонов; сортировка по убыванию — последние изменения сверху. |

## Хранение и «устаревание» строк

В [Настройках](/docs/settings/) задаётся **`results_retention_days`** (глобально для приложения): записи старше порога могут удаляться фоновой логикой или при обслуживании — уточняйте по вашей версии `server.js` / cron. Перед **очисткой из UI** сделайте выборку или экспорт, если нужны исторические цены.

## Связь с сопоставлением

Для [Сопоставление](/docs/matches/) нужны **актуальные** строки в `prices` по выбранным конкурентам. Если матчинг даёт странные пары — сверьте качество **названий** и **SKU** в результатах.

## API
