## Статусы (общая логика)

| Статус | Что обычно значит |
|--------|-------------------|
| `pending` | Ждёт парсинга или возвращён после reset. |
| `ok` / success (как в вашей версии) | Успешно разобрано, результат в `prices`. |
| `error` | Исключение при запросе или разборе — смотрите **сообщение ошибки** в таблице или в логах. |

Точные имена статусов смотрите в колонке UI и в [API](/docs/api/#pages--parse-queue).

## Типовой рабочий цикл

1. **Фильтр по проекту** — выберите конкурента.
2. **Добавьте URL** (по одному, списком или через импорт — как в вашей сборке).
3. **Запустите парсинг** для выбранных строк или для всего отфильтрованного набора (кнопки зависят от UI).
4. Обновите страницу или дождитесь авто-обновления — проверьте статусы.
5. Для `error`: откройте карточку ошибки → исправьте URL или [проект](/docs/projects/) → **Reset** в `pending` → снова парсинг.

## Массовые операции (осторожно)

- **Очистка** по фильтру удаляет строки из очереди — убедитесь, что фильтр не шире, чем нужно.
- **Reset** не удаляет URL, а меняет статус — безопаснее для повторных попыток.

## Добавление URL: вручную, списком, импорт

- **По одному** — подходит для точечной проверки селекторов после правок в [Конкуренты](/docs/projects/).
- **Списком / textarea** — вставка десятков URL с одного домена; проверьте, что каждая строка — полный `https://…` или путь, который нормализует ваш фронт.
- **Импорт из файла** (если есть в вашей сборке) — заранее уберите дубликаты и мусорные редиректы; большие файлы лучше дробить, чтобы не упираться в **таймаут** HTTP и лимиты `default_limit` на отображение.

## Связь с воркером и discover

- **Парсинг** в продакшене часто выполняет отдельный процесс **`worker.js`** (или аналог): если в UI статусы не меняются, а API показывает `pending`, смотрите, запущен ли воркер и совпадает ли он с той же БД, что и `npm start` API.
- **Discover / обход sitemap** (если используете) зависит от лимитов `discover_*` в [Настройках](/docs/settings/): при всплеске URL в очереди сузьте лимиты или включите задержки.

## Связь с другими экранами

- Нет новых строк в [Результаты](/docs/results/) — сначала очередь и статусы.
- Много **error** — сначала селекторы и доступность сайта, не «Мои товары».

## API

Префиксы **`/api/pages`** и **`/api/parse`** — алиасы одного роутера. Список эндпоинтов: [Pages / Parse queue](/docs/api/#pages--parse-queue). Для точечной проверки одной страницы: `POST /api/pages/page/:id`.
