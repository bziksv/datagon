Для изменения лимитов идите в [Настройки](/docs/settings/).

## Когда смотреть этот экран

| Ситуация | Что искать |
|----------|------------|
| Кнопка «Синк» нажата, UI не меняется | Блок **МойСклад** или **глобальный синк** — завершилась ли задача, нет ли ошибки. |
| Долгий матчинг | Блок **Сопоставление** — прогресс или зависание. |
| Парсинг «висит» | Очередь / автообход — не застрял ли воркер. |

## Что обычно показывает экран (по блокам)

- **Глобальная синхронизация** — массовый прогон по всем включённым источникам «Мои сайты»; длительность зависит от объёма и `sync_*` в [Настройках](/docs/settings/). Не запускайте параллельно с тяжёлыми ручными операциями на той же БД без окна обслуживания.
- **МойСклад** — фоновый `POST /api/ms/sync`: статус смотрите здесь и через [`GET /api/ms/status`](/docs/api/#moysklad).
- **Парсинг / очередь** — связано с `worker.js` и таблицей `pages`: если статус в логах «завис», проверьте процесс воркера на сервере (pm2/systemd) и свободное место на диске для временных файлов (если используются).
- **Сопоставление** — соответствует `POST /api/matches/start-matching`; при ошибке текст часто дублируется в ответе API и в stderr Node.

## Чеклист диагностики (кратко)

1. Зафиксировать **время** нажатия кнопки и **имя** процесса на экране.
2. Обновить логи через **15–60 с** — часть задач пишет прогресс пакетами.
3. Открыть **логи сервера** за тот же интервал (stack trace, MySQL deadlock).
4. Для МС/матчинга — вызвать статус через [REST API](/docs/api/) и сравнить с UI.
5. Если задача «бессрочная» — остановка через API (МС/матчинг) или рестарт воркера/Node по политике эксплуатации.

## Как не путать процессы

- **Очередь страниц** (`pages`) — это **URL конкурентов**, не строки `my_products`.
- **Глобальный синк** — массовая операция по всем источникам; может конкурировать по БД с пользовательскими действиями — планируйте время окна.

## Если кажется, что «всё сломалось»

1. Обновите страницу логов через 10–30 с.
2. Проверьте консоль **сервера Node** (stderr) — часто там stack trace, которого нет в UI.
3. Для МС и матчинга — эндпоинты статуса в [REST API](/docs/api/).
