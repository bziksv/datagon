## Поля формы (логика)

| Поле | Обязательность | Комментарий |
|------|----------------|--------------|
| Название проекта | да | Отображается в списках и фильтрах. |
| Домен / URL | да | База для нормализации ссылок в очереди. |
| **Селектор цены** | да | Без него парсинг обычно бессмысленлен. |
| Селектор названия | желательно | Для отчётов и матчинга по названию. |
| Селектор SKU | желательно | Для матчинга по артикулу. |
| «Нет в наличии» / OOS | по ситуации | Зависит от вёрстки конкурента. |

## Как проверить селекторы до массового парсинга

1. Откройте **реальную карточку товара** конкурента в Chrome/Firefox.
2. **Правый клик** по элементу с ценой → «Исследовать».
3. В консоли вкладки **Elements** подберите селектор, который **уникален** для блока цены (избегайте слишком общих `.price` на всю витрину, если на странице их несколько).
4. Скопируйте селектор в проект Datagon, **сохраните** проект.
5. Добавьте **одну** тестовую URL в [Очередь](/docs/queue/), запустите парсинг только для неё, проверьте [Результаты](/docs/results/).

## Частые причины «пустого» парсинга

- Цена подгружается **JavaScript**-ом после загрузки HTML — в ответе `axios` её нет; нужен другой источник данных или отказ от этого конкурента для авто-парсинга.
- Селектор указывает на **скрытый** блок (display:none) или на промо-цену.
- Конкурент отдаёт **403/429** — снизьте частоту запросов в [Настройках](/docs/settings/) (`page_delay_ms`, размеры батчей).

## Редактирование и удаление

- **Сохранение** изменений селекторов не очищает автоматически старые `prices` — имейте в виду смешанные данные до следующего прогона.
- **Удаление проекта** — необратимо для конфигурации; связанные URL в очереди и результаты обсуждайте с администратором БД.

## Примеры селекторов (идеи, не копируйте вслепую)

| Ситуация | Пример подхода |
|----------|------------------|
| Цена в `span` с классом | `.product-price .value` — убедитесь, что на странице один такой блок. |
| Данные в `data-*` | `[data-product-price]` иногда стабильнее классов с хэшем. |
| SKU в таблице характеристик | XPath или узкий CSS по `id` родительского блока — классы `.row` часто ломаются при обновлении витрины. |

Всегда проверяйте на **мобильной** и **десктопной** вёрстке, если конкурент отдаёт разный HTML.

## Кодировки, редиректы, robots

- Если карточка открывается только после **302** на другой хост — добавляйте в очередь **конечный** URL после редиректов.
- **UTF-8** vs кракозябры в названии — смотрите `Content-Type` ответа; при необходимости правьте проект или прокси.
- **robots.txt / 403 на датацентровых IP** — не лечится селектором; обсуждайте обход (прокси, ручной импорт цен) на уровне процесса.
