Описание режимов сопоставления

Режимы можно менять и сравнивать результаты; позже часть логики можно автоматизировать по метрикам качества.

all · Все поля
Сначала строгое совпадение SKU (trim + верхний регистр). Если нет — сравнение по названию с порогом схожести (по умолчанию 85%).
sku · Строгий SKU
Только точное совпадение после trim и приведения к верхнему регистру. Пробелы внутри артикула и Unicode-тире не приводятся к общему виду. Название не используется.
sku_norm · Нормализация
Ключ SKU: NFKC, верхний регистр, удаление пробелов/NBSP, унификация тире в ASCII--. Берётся первая совпавшая строка прайса в текущем порядке выборки (как раньше для строгого режима, но по нормализованному ключу).
sku_best · Нормализация + выбор строки
Тот же ключ, что и в sku_norm, но среди всех строк прайса с этим ключом выбирается одна: сначала большая дата парсинга (MAX(parsed_at) по группе строки), затем стабильный порядок по названию и исходному SKU.
name · Только название
SKU не участвует; только схожесть нормализованных названий при длине > 3 символов (у вас и у позиции конкурента).

Как считает сервер и что уходит в API

Режим сопоставления (сервер)

  • Все поля — сначала строгий SKU (trim + верхний регистр). Если нет пары — шаг по названию (порог по умолчанию 85%).
  • SKU строгий — только trim + верхний регистр; название не используется.
  • SKU нормализация — один ключ для вашего и конкурента: NFKC, верхний регистр, без пробелов/NBSP, тире приведены к -; берётся первое совпадение в порядке строк прайса.
  • SKU нормализация + выбор строки — тот же ключ; среди всех строк с этим ключом выбирается одна по максимальной дате парсинга группы, затем по названию и SKU (стабильно при повторных запусках).
  • Только название — SKU не участвует; длина нормализованного названия > 3 у обеих сторон.

Поле «Умный поиск»

  • Для таблицы «Мои товары» строка поиска уходит в /api/matches/my-products: подстрока ищется в SKU, названии и коде импорта (source_id).
  • Для кнопок «Запустить авто по фильтру» и «Продолжить/повторить» та же строка передаётся как фильтр набора товаров для сопоставления: на сервере отбираются только активные товары вашего сайта, у которых подстрока входит в SKU или название (без source_id). К этому набору затем применяется выбранный режим сопоставления. При «Запустить по выбранным товарам» в задачу попадают только отмеченные в таблице ID; строка «Умный поиск» на состав набора для API не влияет.

Обход конкурентов и их прайсов идёт по очереди; для режима по названию на один ваш товар выбирается не более одной лучшей позиции конкурента среди кандидатов не ниже порога.
