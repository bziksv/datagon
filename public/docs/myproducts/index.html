Мои товары

Экран /my-products.html — основная таблица каталога my_products: что пришло с Моих сайтов, активность, связь с МойСклад (ms_export), сравнение с ценами конкурентов (если настроены парсинг и сопоставление).

Снимок интерфейса. PNG обновляют: npm run docs:capture-screenshots (с DOCS_USER и DOCS_PASSWORD — с живой панели; без входа — с макета /doc-screenshots/myproducts-sample.html) и npm run docs:docusaurus:build. Таблица в кадре — viewport (верх экрана). Подробнее о съёмке.

Структура экрана (обобщённо)​

  1. Верхняя панель фильтров — сайт-источник, активность, связь с МС, аудит сопоставлений, разрыв цены с конкурентом и др. (набор зависит от версии).
  2. Таблица — строки товара; горизонтальный скролл при большом числе колонок (как в эталоне МойСклад в проекте).
  3. Действия по строке — обновление из источника, подтягивание цены с конкурента (если политика включена), прочие кнопки из UI.
  4. Сортировка — клик по заголовку столбца; направление и колонка часто запоминаются на пользователя.

Фильтры: зачем каждый​

ФильтрПрактический смысл
СайтОдин источник (my_sites.id) или все сразу.
Статус (активность)После синка «лишние» позиции могут быть неактивны — см. главная FAQ п.7.
Связь с МСЕсть ли соответствие в ms_export по нормализованному коду (подробности в REST API).
Разрыв с конкурентомДиапазон отклонения цены; может требоваться курс USD/EUR — курсы подтягиваются API (см. fx-rates в справке API).
ПоискНесколько слов через пробел — по полям товара (как на сервере).

Сортировка​

  • По умолчанию часто используется порядок по ID с сайта (возрастание) — уточняйте по подсказке в шапке таблицы.
  • Повторный клик по тому же столбцу меняет направление.
  • Если сортировка «сбрасывается» — проверьте, не открыта ли другая вкладка с тем же пользователем.

Типовые задачи​

Найти товар по артикулу​

Введите SKU (или часть) в поиск, при необходимости сузьте сайт. Проверьте, не включён ли лишний фильтр (например, только «связанные с МС»).

Понять, почему нет цены конкурента​

Нужны результаты парсинга по этому конкуренту и сопоставление — без подтверждённой или автоматической связи колонки конкурента могут быть пустыми. Цепочка: Конкуренты → Очередь → Результаты → Сопоставление.

Обновить одну строку из источника​

Используйте действие обновить одну позицию (название в UI может отличаться) — на сервере это соответствует POST /api/my-products/refresh-one (см. API).

Производительность​

Список и статистика (/api/my-products/stats) кэшируются на короткий TTL — при массовых изменениях цифры могут обновиться с задержкой. При «тяжёлых» запросах в MySQL см. FAQ гл. п.10.

Действия по строке (что обычно есть в UI)​

ДействиеКогда применять
Обновить из источникаОдна позиция «поехала» после правки в БД магазина; не ждите полного синка.
Подтянуть цену конкурентаЕсли уже есть сопоставление и политика позволяет точечный запрос к данным prices.
Перейти к связанным даннымЗависит от версии: иногда ссылка на результат парсинга или на карточку МС.

Точные имена кнопок и доступность — по подсказкам в интерфейсе; дублирующие вызовы смотрите в REST API — My products.

Фильтры «аудит» и отчётность​

Фильтры вроде «разрыв с конкурентом», «нет связи с МС», «неподтверждённое сопоставление» предназначены для еженедельного обзора: сохраните набор фильтров (если UI запоминает) и проходите список пакетами, а не всем каталогом сразу.
