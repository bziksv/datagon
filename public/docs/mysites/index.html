Мои сайты

Раздел /my-sites.html — учёт источников собственного каталога: подключение к внешней MySQL (магазин на Bitrix, Webasyst и т.д.), проверка чтения полей и синхронизация данных в локальную таблицу Datagon my_products. Без рабочих источников неполноценны «Мои товары», сопоставление и часть отчётов по конкурентам.

Снимок интерфейса. PNG обновляют: npm run docs:capture-screenshots (с DOCS_USER и DOCS_PASSWORD — с живой панели; без входа — с макета /doc-screenshots/mysites-sample.html) и npm run docs:docusaurus:build. Для этого экрана — полная страница в кадре (см. съёмка).

Перед началом​

Убедитесь, что:

  • с сервера Datagon (или вашей машины, если так настроено) есть сетевой доступ к хосту БД источника (firewall, VPN, bind-address MySQL);
  • у пользователя БД есть SELECT (и при необходимости другие права) на таблицу товаров и связанные таблицы (SKU для Webasyst);
  • вы знаете точное имя таблицы и имена колонок для SKU, названия, цены, остатка, признака активности.

Пошагово: новый источник​

  1. Откройте Мои сайты → кнопка добавления / форма нового подключения (зависит от версии UI).
  2. Заполните хост, порт (часто 3306), имя базы, пользователь, пароль.
  3. Укажите таблицу товаров и сопоставьте поля (SKU, название, цена, валюта, остаток и т.д.) — используйте подсказки на форме.
  4. Для Webasyst: отдельно укажите таблицу SKU и поля wa_*, если это требуется вашей схемой.
  5. Нажмите сохранить / проверить подключение (если есть). При ошибке читайте текст: «DB Connect», «Access denied», «Unknown database».
  6. Запустите превью или тестовую выборку (если доступно) — убедитесь, что в превью осмысленные строки.
  7. Запустите синхронизацию (полную или поэтапную — см. подсказки). Длительные прогоны смотрите в Логи.

Что происходит при синхронизации (логика)​

Перед полным прогоном часто выполняется подготовка: позиции помечаются неактивными, затем найденные во внешней БД обновляются и снова помечаются активными. Позиции, которых нет в источнике после прохода, могут остаться неактивными — это ожидаемо (см. FAQ на главной про «неактивны после синка»).

Типовые ошибки​

СимптомЧто проверить
DB Connect / timeoutсеть, порт, хост, что MySQL слушает внешние подключения
Access deniedлогин/пароль, хост пользователя (user@'%')
Unknown table / columnимя таблицы и маппинг полей
Пустое превьюфильтр в запросе, пустая таблица, неверная схема

Bitrix и Webasyst: что чаще всего мапят​

НазначениеBitrix (ориентир)Webasyst (ориентир)
Таблица товаровчасто b_catalog_product + b_catalog_price / поля в формесвоя схема shop_product и SKU-таблица
SKU / артикулARTICLE, XML_ID — зависит от настройки магазинаполя wa_* в форме
Цена / валютаиз каталога / типа ценыиз SKU или товара — см. подсказки в UI

Точные имена колонок снимите запросом SHOW COLUMNS / ER-диаграммой — не полагайтесь на чужие примеры из интернета.

Безопасность и эксплуатация​

  • Пользователь БД для чтения лучше иметь только SELECT на нужные таблицы и без FILE/SHUTDOWN.
  • Пароль источника хранится в БД Datagon — ограничьте доступ к бэкапам и к админам приложения.
  • Долгий синк на «боевом» источнике — планируйте на низкую нагрузку; тяжёлые JOIN в настройках источника увеличивают блокировки на стороне магазина.

Связь с другими разделами​

  • Мои товары — просмотр и действия по строкам my_products.
  • Сопоставление — нужны актуальные «наши» SKU.
  • REST API — My sites — GET/POST/PUT/DELETE /api/my-sites, …/sync, sync-all-real.
