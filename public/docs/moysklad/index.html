МойСклад

Экран /moysklad.html — работа с таблицей ms_export: данные, подтянутые из API МойСклад, фильтры по типу номенклатуры, поиск по строкам экспорта, запуск и контроль фоновой синхронизации, вспомогательные действия (пересборка кэшей связей — если доступно в вашей сборке).

Снимок интерфейса. PNG обновляют: npm run docs:capture-screenshots (с DOCS_USER и DOCS_PASSWORD — с живой панели; без входа — с макета /doc-screenshots/moysklad-sample.html) и npm run docs:docusaurus:build. Для этой таблицы — viewport (видимая область окна). Подробнее о съёмке.

С чего начать новому пользователю​

  1. Убедитесь, что в config.js или в переменной окружения MS_TOKEN задан рабочий токен с правами на чтение нужных сущностей МС.
  2. Откройте МойСклад и проверьте статус последней задачи (не висит ли ошибка с прошлого запуска).
  3. При необходимости нажмите Синхронизация и дождитесь завершения (индикатор в UI + Логи).
  4. Используйте поиск и фильтры, чтобы найти конкретный код/артикул и сверить поля с вашей учётной политикой.

Блоки интерфейса​

БлокНазначение
Статус / прогрессОтображение фоновой задачи синка с API МС; текст ошибки при сбое.
ФильтрыТип (Товар / Комплект / «все»), архив, остатки, «на сайте», строка поиска.
ТаблицаСтроки ms_export; широкая таблица со скроллом; настройка видимых колонок (если есть).
ДействияЗапуск/остановка синка, пересборка кэшей (при наличии).

Типовой сценарий сверки с «Моими товарами»​

В Мои товары признак связи с МС опирается на сопоставление нормализованных кодов между my_products и ms_export. Если связь «плавает»:

  • выполните повторный sync МС;
  • проверьте, что в МС и у вас в источнике одинаковая логика артикула (пробелы, регистр нивелируются на сервере — см. API);
  • при массовых правках — кэш статистики может задержаться на TTL.

Если синхронизация не идёт​

  1. Логи — не застряла ли предыдущая задача.
  2. Токен и лимиты API МС (ошибки 401/403 в логах сервера).
  3. GET /api/ms/status — вручную через API (см. документацию API).
  4. После смены токена в конфиге — перезапуск Node.

Колонки ms_export (общий смысл)​

Набор колонок зависит от версии выгрузки; ориентиры:

ТемаЧто смотреть
Код / артикулСвязь с Мои товары после нормализации.
ТипТовар / комплект — комплекты часто требуют отдельной бизнес-логики в отчётах.
Остаток, ценыСверка с витриной МС; расхождения — часовой пояс и налоги.
«На сайте» / архивФильтры в UI; помогают отсечь неактуальные строки при ручной проверке.

Лимиты API МойСклад​

У МС есть rate limit и размер страницы; в Datagon см. ms_sync_page_limit и ms_sync_delay_ms в Настройках. При 429 или таймаутах — увеличьте паузу, уменьшите страницу, не запускайте второй синк, пока первый не завершён.

API (для автоматизации)​

  • POST /api/ms/sync — старт фона.
  • GET /api/ms/status — состояние.
  • GET /api/ms/export — страницы таблицы с query search, type, limit, offset.

Полное описание — REST API — MoySklad.
