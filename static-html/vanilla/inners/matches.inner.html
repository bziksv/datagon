Сопоставление
Запуск сопоставления и таблица совпадений: сайт для задачи и сайт для списка настраиваются отдельно.
Гайд Как проходить страницу по шагам

Рекомендуемый порядок: сначала SKU-прогон с «Нормализация + детерминированная строка», при необходимости — «Нормализация артикула», затем «Строгий отбор»; после этого — «Только название» и ручная доводка. Так вы быстрее закрываете очевидные пары и не смешиваете разные типы ошибок в одной куче.

Что подставляется при открытии страницы
  1. Параметры в адресе (query) имеют приоритет: my_site_id, list_my_site_id, list_competitor_id, status, mode, product_search, matches_page, matches_limit и др. Если параметра в URL нет — может подставиться значение из сохранённых фильтров (следующий пункт).
  2. Сохранённые фильтры в браузере (модуль фильтров, localStorage): список «Совпадений», конкурент, умный поиск, статус и лимит — только для тех полей, которых нет в query; сайт запуска по умолчанию — из контекста «Мои товары», если нет my_site_id.
  3. Вид карточек и гайда — в localStorage под префиксом datagon_matches_* (развёрнутость гайда, шагов 1–4, «Мои товары», ширины колонок). На сопоставление товаров не влияет.
  4. Ручное сопоставление (шаг 3): поиск в прайсе и «Подтвердить вручную» — только в блоке под строкой очереди по кнопке «Найти по конкуренту»; очередь зависит от «Мой сайт (список)» и конкурента в «Совпадениях».
Шаг 1
Параметры запуска

Выберите мой сайт (запуск), отметьте конкурентов, при необходимости сузьте набор через умный поиск и пакетные параметры. Для первого прохода по артикулам рекомендуем режим «Нормализация + детерминированная строка» — он устойчив к типичным расхождениям в SKU и к нескольким строкам прайса с одним ключом. При необходимости затем прогоните «Нормализация артикула», затем «Строгий отбор (trim + регистр)», чтобы сузить или перепроверить остатки. Нажмите «Запустить авто по фильтру» или отметьте товары в таблице «Мои товары» и запустите по выбранным.

Дождитесь завершения задачи по логам. Затем смените режим на «Только название», снова запустите сопоставление по тому же или расширенному набору — так вы отдельно обрабатываете артикулы и названия, без смешения логики в одном прогоне. Ожидающие строки в «Совпадениях» по разным режимам (SKU / название / нормализации) для тех же конкурентов сохраняются; заново снимаются только ожидающие по выбранным конкурентам при режиме запуска «все поля».

В статусе задачи числитель/знаменатель — пары «наш товар × конкурент» (шаги воркера), а не число URL в очереди парсинга прайса.

Шаг 2
Совпадения

В блоке «Совпадения» выберите «Мой сайт (список)» и при необходимости конкурента, отфильтруйте статус «Ожидает», примените фильтры. Просмотрите каждую строку: верные пары подтверждайте, сомнительные или неверные — отклоняйте (они попадут в ручную очередь и не будут снова автоматически предлагаться для этого конкурента, пока вы их не уберёте из очереди).

После полного прохода по результатам SKU вернитесь к шагу 1, переключите режим на название, снова запустите задачу и повторите тот же цикл в этом блоке для новых строк «Ожидает».

Шаг 3
Ручное сопоставление

Очередь шага 3 — не только отклонения из шага 2: сюда попадают и товары с отклонённым авто-совпадением, и те, по которым при последнем прогоне с шага 1 авто не нашло пару (оба варианта пишутся в исключения, чтобы не гонять их снова в авто). Для выбранного конкурента они исключены из следующих автоматических прогонов по SKU и названию, пока запись не удалена из очереди. В таблице колонка «Причина» различает эти два случая (подписи на бейджах; при наведении — развёрнутое пояснение). Кнопка «События» — таблица последних событий по товару сразу под строкой очереди.

Используйте те же «Мой сайт (список)» и фильтр «Конкурент» в «Совпадениях» для очереди. Кнопка «Найти по конкуренту» в строке открывает под ней поиск в прайсе, подстановку SKU/названия из результата и кнопку «Подтвердить вручную» — так создаётся запись «Вручную» и строка снимается с очереди. Кнопка «Вернуть в авто» убирает строку из очереди без фиксации пары — товар снова участвует в авто-сопоставлении.

Кнопка «В архив» переносит ситуацию на шаг 4, если вы осознанно отказываетесь от ручной привязки после разбора.

Шаг 4
Архив: отказ после ручного сопоставления

Сюда попадают только позиции после ручного отказа из шага 3. Записи можно удалить — товар снова сможет участвовать в авто-сопоставлении. Этот блок для «хвоста», который вы решите обработать позже отдельной политикой.

Описание режимов сопоставления

Режимы можно менять и сравнивать результаты; позже часть логики можно автоматизировать по метрикам качества.

all · Все поля
Сначала строгое совпадение SKU (trim + верхний регистр). Если нет — сравнение по названию с порогом схожести (по умолчанию 85%).
sku_best · Нормализация + выбор строки
Тот же ключ, что и в sku_norm, но среди всех строк прайса с этим ключом выбирается одна: сначала большая дата парсинга (MAX(parsed_at) по группе строки), затем стабильный порядок по названию и исходному SKU.
sku_norm · Нормализация
Ключ SKU: NFKC, верхний регистр, удаление пробелов/NBSP, унификация тире в ASCII--. Берётся первая совпавшая строка прайса в текущем порядке выборки (как раньше для строгого режима, но по нормализованному ключу).
sku · Строгий SKU
Только точное совпадение после trim и приведения к верхнему регистру. Пробелы внутри артикула и Unicode-тире не приводятся к общему виду. Название не используется.
name · Только название
SKU не участвует; только схожесть нормализованных названий при длине > 3 символов (у вас и у позиции конкурента).

Как считает сервер и что уходит в API

Режим сопоставления (сервер)

  • Все поля — сначала строгий SKU (trim + верхний регистр). Если нет пары — шаг по названию (порог по умолчанию 85%).
  • SKU нормализация + выбор строки — тот же ключ, что у нормализации; среди всех строк с этим ключом выбирается одна по максимальной дате парсинга группы, затем по названию и SKU (стабильно при повторных запусках). Рекомендуемый первый прогон по артикулам.
  • SKU нормализация — один ключ для вашего и конкурента: NFKC, верхний регистр, без пробелов/NBSP, тире приведены к -; берётся первое совпадение в порядке строк прайса.
  • SKU строгий — только trim + верхний регистр; название не используется.
  • Только название — SKU не участвует; длина нормализованного названия > 3 у обеих сторон.

Поле «Умный поиск»

  • Для таблицы «Мои товары» строка поиска уходит в /api/matches/my-products: подстрока ищется в SKU, названии и коде импорта (source_id).
  • Для кнопок «Запустить авто по фильтру» и «Продолжить/повторить» та же строка передаётся как фильтр набора товаров для сопоставления: на сервере отбираются только активные товары вашего сайта, у которых подстрока входит в SKU или название (без source_id). К этому набору затем применяется выбранный режим сопоставления. При «Запустить по выбранным товарам» в задачу попадают только отмеченные в таблице ID; строка «Умный поиск» на состав набора для API не влияет.

Обход конкурентов и их прайсов идёт по очереди; для режима по названию на один ваш товар выбирается не более одной лучшей позиции конкурента среди кандидатов не ниже порога.

Продолжить/повторить — новая задача по последней остановленной, сбойной или отменённой (если таких не было — по последней завершённой): подставляются сохранённые параметры и чекпоинт, список товаров снова загружается, но уже сопоставленные с выбранными конкурентами позиции пропускаются (см. в логе «Продолжение: пропущено…»). Счётчики справа и числитель пар «наш товар × конкурент» (знаменатель ≈ ваши товары в задаче × отмеченные конкуренты; не объём очереди парсинга) при продолжении наращиваются с предыдущей задачи. «Новых записей в „Совпадениях“» — только новые строки в БД за этот прогон.

Ручное сопоставление Шаг 3

Одна общая очередь на две ситуации (см. колонку «Причина»): после «Отклонить» в совпадениях (шаг 2) и после авто-прогона (шаг 1), если для этого конкурента не найдено пары. Это не ошибка — так товар не участвует в следующем авто, пока вы не разберёте строку или не нажмёте «Вернуть в авто». Подпись «Авто не нашло пару» ставится только после прогона с шага 1, а не после «Отклонить». Ручная фиксация (под строкой очереди) создаёт строку «Подтверждено» с типом «Вручную».

Колонка «Причина»: «Авто-совпадение вы отклонили» — в шаге 2 «Совпадения» для этой строки было предложение от авто-сопоставления, но вы нажали «Отклонить»; «Авто не нашло пару» — в последнем прогоне задачи пара для этого конкурента не найдена. Наведите курсор на бейдж — откроется пояснение.

Фильтры «Мой сайт (список)» и «Конкурент» берутся из блока «Совпадения» выше (при смене нажмите «Обновить очередь»). Селект «Причина» ниже ограничивает очередь по значению колонки «Причина» (или показывает все).
